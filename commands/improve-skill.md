# Improve Skill

기존 스킬의 효과를 분석하고, 내용을 개선합니다.

## 절차

### 1. 스킬 목록 조회

다음 위치에서 스킬 파일을 수집합니다:

- ~/.claude/skills/auto/*.md (글로벌 자동 스킬)
- {현재 프로젝트}/.claude/skills/auto/*.md (프로젝트 자동 스킬)
- ~/.claude/growth/skill-registry.json (메타데이터)

스킬이 없으면 "아직 자동 생성된 스킬이 없습니다. /extract-skill로 먼저 스킬을 생성하세요."라고 안내합니다.

### 2. 개선 대상 선정

사용자에게 개선할 스킬을 선택하게 합니다. 또는 다음 기준으로 자동 추천:

- 최근 사용 시 실패하거나 수정이 필요했던 스킬
- times_used가 높지만 절차가 부실한 스킬
- 생성된 지 오래되어 현재 코드베이스와 맞지 않는 스킬

### 3. 개선 분석

선택된 스킬에 대해:

- 현재 스킬 내용 읽기
- 관련 코드/프로젝트 상태와 비교
- 빠진 단계, 잘못된 경로, 오래된 명령어 식별
- 개선안 작성

### 4. 개선 적용

사용자 승인 후:

- 스킬 파일 내용 업데이트
- skill-registry.json의 메타데이터 업데이트
- ~/.claude/growth/improvement-log.md에 개선 이력 기록:

```
### YYYY-MM-DD Skill Improvement
- 스킬: {skill-name}
- 변경: {변경 요약}
- 이유: {개선 이유}
```

## 참고

- 스킬의 핵심 목적은 유지하면서 절차만 개선합니다
- 근본적으로 다른 패턴이 필요하면 새 스킬 생성을 권장합니다
