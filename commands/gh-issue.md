# GitHub Issue 등록

사용자가 설명한 문제를 GitHub 이슈로 등록합니다.

## 사용법

```
/gh-issue [이슈 설명]
```

## 지침

1. 사용자가 제공한 설명을 분석하여 다음 정보를 추출합니다:
   - 문제 상황 (Problem)
   - 현재 동작 (Current Behavior)
   - 기대 동작 (Expected Behavior)
   - 관련 컴포넌트 (힌트가 있는 경우)

2. 이슈 제목을 간결하고 명확하게 작성합니다 (한글 또는 영어)

3. 이슈 본문을 다음 구조로 작성합니다:

```markdown
## 문제

[문제 설명]

## 현재 상태

- [현재 동작 설명]

## 기대 동작

- [기대하는 동작 설명]

## 관련 컴포넌트

- [관련 파일/컴포넌트 경로]

## 추가 정보

[스크린샷이나 추가 컨텍스트가 있으면 기재]
```

4. `gh issue create` 명령어로 이슈를 생성합니다:

```bash
gh issue create --title "제목" --body "본문"
```

5. 생성된 이슈 URL을 사용자에게 반환합니다.

## 옵션

- 라벨이 필요하면 `--label` 옵션 추가
- 담당자 지정이 필요하면 `--assignee` 옵션 추가

## 예시

사용자: `/gh-issue 로그인 버튼이 모바일에서 너무 작아요`

이슈 생성:
- 제목: "모바일 로그인 버튼 크기 개선"
- 본문: 문제 설명, 현재 상태, 기대 동작 포함
