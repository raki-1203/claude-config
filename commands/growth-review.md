# Growth Review

사용자의 최근 작업을 분석하여 에이전트 시스템을 성장시킵니다. 아래 단계를 순서대로 수행하세요.

## 1. 세션 로그 분석

~/.claude/growth/session-log.jsonl 파일을 읽어 최근 7일간의 세션 통계를 분석합니다.

- 프로젝트별 세션 수, 도구 사용 패턴
- 파일이 없거나 비어있으면 "아직 세션 로그가 축적되지 않았습니다"라고 알리고 2단계로 넘어감

## 2. 스킬 후보 탐색

최근 대화와 작업 패턴에서 반복되는 작업을 식별합니다.

- 사용자에게 "최근 반복적으로 수행한 작업이 있나요?"라고 질문
- 현재 세션의 도구 호출 패턴에서 반복 시퀀스 식별
- 3회 이상 반복된 패턴 → 스킬 후보로 제안
- 스킬 후보별로 범용/프로젝트 특화 여부를 판단:
  - 범용 → ~/.claude/skills/auto/에 SKILL.md 생성
  - 프로젝트 특화 → 해당 프로젝트의 .claude/skills/auto/에 생성
- 반드시 사용자 승인 후 생성
- ~/.claude/skills/templates/skill-template.md 포맷을 사용

## 3. 기존 스킬 점검

~/.claude/skills/auto/ 의 기존 스킬을 검토합니다.

- 각 스킬의 times_used 카운터 확인
- 30일 이상 미사용 스킬 → 아카이브 또는 삭제 제안
- 내용이 부실한 스킬 → 개선 제안

## 4. 메모리 점검

~/.claude/projects/*/memory/ 디렉토리들을 스캔합니다.

- 오래된/중복된 메모리 식별 → 정리 제안
- 프로젝트 간 공통 피드백 패턴 확인

## 5. CLAUDE.md 규칙 진화

프로젝트별 feedback 메모리에서 반복 패턴을 확인합니다.

- 2개 이상 프로젝트에 걸친 동일한 피드백 → ~/CLAUDE.md 승격 후보 제안
- 기존 ~/CLAUDE.md 규칙 중 더 이상 유효하지 않은 것 식별
- 변경 사항은 반드시 사용자 승인 후 적용

## 6. 성장 리포트

작업 결과를 요약하고 ~/.claude/growth/improvement-log.md에 기록합니다.

```
## YYYY-MM-DD Growth Review

### 세션 통계
- 분석 기간: ...
- 총 세션 수: ...
- 주요 프로젝트: ...

### 스킬 변경
- 생성: ...
- 개선: ...
- 아카이브: ...

### 규칙 변경
- 승격: ...
- 수정: ...

### 메모리 정리
- 정리: ...
```

모든 변경은 사용자 승인 후에만 적용합니다.
