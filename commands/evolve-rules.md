# Evolve Rules

프로젝트별 메모리에서 반복 피드백을 분석하고, 글로벌 규칙으로 승격할 후보를 제안합니다.

## 절차

### 1. 피드백 수집

모든 프로젝트의 메모리에서 피드백을 수집합니다:

- ~/.claude/projects/*/memory/feedback_*.md 스캔
- 각 프로젝트의 .claude/MEMORY.md도 확인
- 피드백 내용, 프로젝트, 날짜를 정리

### 2. 패턴 분석

수집된 피드백에서 반복 패턴을 식별합니다:

- 2개 이상 프로젝트에서 동일한 피드백 → 승격 후보
- 단일 프로젝트 피드백이지만 보편적으로 적용 가능한 것 → 후보로 표시

### 3. 현재 규칙 대조

~/CLAUDE.md의 기존 규칙과 대조합니다:

- 이미 규칙으로 존재하는 피드백 → 스킵
- 기존 규칙과 충돌하는 피드백 → 사용자에게 질문
- 새로운 패턴 → 승격 후보로 제안

### 4. 승격 제안

승격 후보를 사용자에게 제시합니다:

```
## 규칙 승격 후보

### 1. {규칙 이름}
- 근거: {프로젝트 A}의 feedback_xxx.md, {프로젝트 B}의 feedback_yyy.md
- 제안 규칙: "{규칙 내용}"
- 추가 위치: ~/CLAUDE.md의 "{섹션}" 아래
```

### 5. 적용

사용자 승인 후:

- ~/CLAUDE.md에 규칙 추가
- 원본 프로젝트 피드백 메모리에 "~/CLAUDE.md로 승격됨 (날짜)" 메모 추가
- ~/.claude/growth/improvement-log.md에 기록

## 이전 사례

kt-innovation-hub 프로젝트에서 다음 피드백이 ~/CLAUDE.md로 승격된 적 있음:
- no-fake-pass, show-logs, pr-template, review-tone, no-dev-server, uv-run (2026-03-21)

이 패턴을 참고하여 자연스럽게 승격 프로세스를 진행합니다.
