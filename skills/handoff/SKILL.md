# Handoff - 에이전트 간 컨텍스트 인계

컨텍스트가 소진되거나 작업을 다음 에이전트에게 넘길 때 HANDOFF.md를 작성합니다.

## 절차

1. 프로젝트 루트에 기존 HANDOFF.md가 있는지 확인
2. 있으면 먼저 읽어서 이전 컨텍스트 파악
3. HANDOFF.md를 생성 또는 업데이트:

```markdown
# Handoff Document

## 목표
현재 작업의 최종 목표

## 현재 진행 상황
- 완료된 작업 목록
- 현재 상태 (어디까지 진행됨)

## 성공한 접근 방법
- 효과가 있었던 방법과 이유

## 실패한 접근 방법
- 시도했지만 실패한 방법과 이유 (반복 방지)

## 다음 단계
- 구체적인 액션 아이템
- 참고할 파일 경로
- 주의사항

## 관련 파일
- 수정된 파일 목록
- 참고해야 할 파일 목록
```

4. 파일 경로를 사용자에게 알려주어 새 세션에서 해당 경로만 입력하면 이어서 작업 가능하게 함

## 사용 예시

```
/handoff
```

새 세션에서 이어받기:
```
./HANDOFF.md
```
